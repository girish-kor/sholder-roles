# Contributing to @sholder/roles

Thanks for your interest in contributing. This document outlines the minimal steps to get changes accepted quickly.

Getting started

- Fork the repo and create a feature branch from `main`.
- Run `npm install` and `npm run build` before running tests.

Code style

- Follow the existing TypeScript style. Run `npm run format` and `npm run lint`.

Testing

- Unit tests: `npm test`
- For an interactive test runner: `npm run test:watch`

Pull request checklist

- Ensure lint and tests pass locally.
- Add tests for new behavior or bug fixes.
- Keep changes small and focused; one logical change per PR.

Release process

- Releases follow semantic versioning. Tag and prepare changelog entries when proposing a release.

Reporting issues

- Use the issue tracker to report bugs or request features. Provide a minimal reproduction when possible.

Contact

- Open a PR or discussion; maintainers will review in priority order.
